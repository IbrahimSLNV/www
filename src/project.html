NAVIGATION DE NAYER

Hi, I'm Ibrahim Selmanov, first years student in Electronics-ICT @ Thomas More Campus De Nayer

My project, made with two other students, is a website containing maps of Campus De Nayer. Users are able to dynamically navigate these maps and click on different locations to show their details.

SOFTWARE

Due to issues with space, you can use the different buttons to navigate between each explanation of my code.

Vue.js is a JavaScript framework used to build websites and web apps. It helps you create interactive parts of a webpage more easily (like buttons that do things when you click them, or content that updates without reloading the page). Components in Vue (and even other frameworks) are like lego's. Think of them as reusable bits of code that represent parts of your webpage (like a navigation bar, a button, or a comment section). Each component has its own HTML, CSS, and JavaScript, so it handles its own look and behavior. So with Vue, instead of writing one huge file for everything, you build your page out of these small, organized pieces.

Now you may ask how I implemented Vue's components inside my project? Click on the ">" button.

Using the explanation from earlier, here I used three main components: Welcome.vue, Navigation.vue and RoomDetails.vue. Each of these components represents a different section of the website. The welcome component is a starting screen that appears when the page loads for the first time. The navigation component lets the user navigate our maps. And the RoomDetails component appears each time the users selects a room. Only one component is shown at a time, depending on which variables are true: showWelcome, showNavigation or showRoomDetails. To make navigating between each components smoother, I added transitions using the <Transitions> tag. This tag is pre-built in Vue.js and adds a visual effect.

Now this could be the most important piece of code inside my main file. The getRoomData(name) function is used to get information about a room when a user clicks on it. It sends a request to a server using Axios, asking for room data based on the name that was passed to the function. For example, if the name is "A0.04", the function tries to get data from http://localhost:5000/rooms/name/A004. If the server responds successfully, the data is saved into a variable called RoomData, and the view is updated: the welcome screen and navigation are hidden, and the RoomDetails component is shown to display the info.

If something goes wrong (like the room doesn't exist or the server is offline) the function still shows the RoomDetails component, but fills it with “waiting” so that the user knows something is loading or missing. This helps the site continue working smoothly, even if there's a problem getting the real data.
